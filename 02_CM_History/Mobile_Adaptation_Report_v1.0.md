# 移动端适配说明 v1.0

## 1. 适配目标

为满足变更需求中“支持手机端查看和操作”的要求，当前演示版系统已完成移动端基础适配，目标是保证企业填报、审核查看和统计查询在手机浏览器中可以完整访问、清晰展示、连续操作。

本次适配定位为课程演示版的移动端兼容方案，不追求原生 App 体验，而是确保在常见竖屏手机分辨率下具备可读性和可操作性。

## 2. 已完成的移动端适配内容

### 2.1 页面结构适配

1. 在窄屏宽度下，系统头部、侧边导航、主工作区改为单列布局。
2. 原桌面端双栏内容区在移动端自动切换为纵向堆叠，避免卡片被压缩。
3. 登录注册区、企业信息表单、就业失业填报表单在手机端均按单列排列显示。

### 2.2 导航适配

1. 工作台左侧导航在移动端自动转换为横向滚动导航条。
2. 不同角色登录后仍然只展示本角色可见功能入口，减少小屏信息干扰。
3. 当前激活页面仍使用高亮按钮标识，保证用户知道自己所在模块。

### 2.3 表单适配

1. 注册页面根据身份动态展示字段，在移动端改为纵向排列，避免字段拥挤。
2. 企业信息、就业填报、失业填报中的按钮区支持自动换行。
3. 新增的查询统计筛选表单在移动端仍可逐项筛选和提交。

### 2.4 表格与数据适配

1. 企业列表、记录列表、审核队列、查询结果表均包裹横向滚动容器。
2. 在手机端无法完整容纳的表格列可横向滑动查看，避免数据被截断。
3. 审核时间线采用卡片列表形式展示，替代表格化信息，便于手机端阅读。

### 2.5 交互细节适配

1. 登录/注册切换按钮在小屏下可拉伸为整行，点击区域更明确。
2. 各类操作按钮在 640px 以下自动扩展为整行布局，减少误触。
3. 统计概览卡片在手机端降为单列显示，保证数值信息足够醒目。

## 3. 当前移动端可支持的典型场景

1. 企业填报人员使用手机登录后，查看本企业信息并录入就业、失业数据草稿。
2. 企业填报人员查看退回意见后重新提交报表。
3. 市级或省级审核人员在手机端查看审核队列与时间线留痕。
4. 管理员或审核人员在手机端查看统计概览与查询筛选结果。

## 4. 当前限制说明

1. 课程演示版仍以浏览器访问为主，未单独开发原生移动端应用。
2. 宽表格虽然支持横向滚动，但在极窄屏设备上阅读效率仍低于桌面端。
3. 当前未引入拍照上传、消息推送、离线缓存等手机特有能力。

## 5. 答辩说明建议

答辩时可按以下口径说明移动端实现：

1. 本系统未单独做 App，而是采用响应式 Web 方式兼容手机端。
2. 手机端重点保证登录注册、数据填报、审核查看、统计查询四类核心流程可用。
3. 对于复杂表格，系统采用横向滚动；对于流程留痕，采用卡片化时间线展示。
4. 若后续继续扩展，可进一步增加手机端专用导航、底部操作栏和截图证据页。
