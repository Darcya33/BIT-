# 云南省企业就业失业数据采集系统 配置管理历史说明

## 1. 文档目的

本文档用于说明课程项目在实施过程中的配置管理策略、版本演进过程、关键基线、变更控制路径以及当前交付状态，作为 02_CM_History 目录的核心说明文件。

## 2. 配置管理范围

本项目纳入配置管理的主要配置项如下：

1. 项目计划文件：01_Project_Plan 目录下的项目计划、技术设计、迭代调整分析、甘特任务表。
2. 变更控制文件：03_Change_Requests 目录下的正式变更单。
3. 系统源码文件：04_Source_Code 目录下的前端、后端代码与运行说明。
4. 仓库级配置：.gitignore、Git 提交历史、远端仓库配置。

## 3. 配置管理工具与策略

- 版本控制工具：Git
- 远端仓库：GitHub
- 默认分支：main
- 配置管理策略：以迭代为单位形成可回溯提交，每个阶段至少保留一条清晰提交记录。
- 工作约束：每轮任务完成后清理缓存文件、临时数据库、辅助运行产物，再执行提交与推送。

## 5. 基线划分

### BL-01 仓库初始化基线

- 内容：代码仓库初始化、.gitignore、生效的远端仓库配置。
- 目标：保证后续文档与代码具备统一版本承载空间。

### BL-02 项目计划基线

- 内容：Project_Plan_v0.1、Project_Plan_v0.2、Gantt_Task_Table_v0.2。
- 目标：形成初始范围、角色分工、时间安排与阶段目标。

### BL-03 架构原型基线

- 内容：Technical_Design_v0.1、前后端骨架代码、README 初版。
- 目标：完成最小可运行系统骨架。

### BL-04 变更控制基线

- 内容：CR-001、CR-002、Iteration_Adjustment_Analysis_v0.1、Project_Plan_v0.3、Gantt_Task_Table_v0.3。
- 目标：把新增需求正式纳入范围和计划控制。

### BL-05 业务流程基线

- 内容：多角色登录、企业管理、就业失业填报、企业上报、市级审核。
- 目标：形成业务主路径第一版。

### BL-06 审核闭环基线

- 内容：省级审核、退回意见展示、退回后重新提交、移动端基础适配。
- 目标：形成课程演示所需的核心业务闭环。

## 6. 变更控制过程

本项目采用如下变更控制流程：

1. 提出变更：由需求方提出正式变更内容和时间点。
2. 影响分析：分析对数据模型、页面、流程、计划的影响。
3. 形成文档：生成 Change Request 文档并归档。
4. 调整计划：同步修订项目计划和甘特任务表。
5. 实施变更：在代码中完成对应开发。
6. 验证结果：通过接口验证和流程联调确认变更已落地。
7. 提交留痕：通过 Git 提交与远端同步保留版本证据。

## 7. 关键变更说明

### CR-001 半月报规则调整

- 提出时间：2026-04-03
- 变更内容：1 至 3 月按半月报填报，其他月份按月报填报。
- 影响：改变报表周期模型、录入规则、校验逻辑。
- 落地结果：已在后端规则校验与前端录入逻辑中实现。

### CR-002 审核流程与移动端扩展

- 提出时间：2026-04-10
- 变更内容：增加企业上报、市级审核、省级审核，并要求手机端可操作，同时延长项目周期。
- 影响：扩大范围，原计划升级为 v0.3 阶段式推进。
- 落地结果：已实现企业上报、市级审核、省级审核与退回重提的第一版闭环，前端完成基础响应式适配。
