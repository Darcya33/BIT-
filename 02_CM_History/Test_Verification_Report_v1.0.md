# 系统联调与验证记录 v1.0

## 1. 文档目的

本文档用于记录课程项目当前版本已经完成的主要联调验证结果，作为课程提交时的技术验证说明文件。本文档重点证明系统核心业务闭环已经打通，不替代完整测试报告。

## 2. 验证范围

本次验证覆盖以下内容：

1. 后端服务可启动并返回健康检查结果。
2. 多角色登录能力可用。
3. 企业信息管理功能可用。
4. 就业与失业数据录入功能可用。
5. 半月报与月报规则校验可用。
6. 企业上报、市级审核、省级审核流程可用。
7. 省级退回后企业重新提交链路可用。

## 3. 验证环境

- 操作系统：Windows
- 后端：FastAPI + SQLite
- 前端：原生 HTML + CSS + JavaScript
- Python 环境：项目本地虚拟环境 .venv
- 验证方式：接口联调 + 页面逻辑验证

## 4. 关键验证结果

### 4.1 服务启动与健康检查

验证结果：通过

说明：

1. 后端服务可在本地端口启动。
2. /health 接口可返回正常状态。

### 4.2 多角色登录

验证结果：通过

覆盖角色：

1. admin
2. enterprise_demo
3. city_reviewer
4. province_reviewer

说明：不同角色登录后可进入对应工作区或执行对应接口。

### 4.3 企业信息管理

验证结果：通过

说明：

1. 管理员可查看企业数据。
2. 企业用户可查看和维护本企业信息。

### 4.4 就业与失业数据录入

验证结果：通过

说明：

1. 企业用户可新增就业记录。
2. 企业用户可新增失业记录。
3. 新增记录以草稿状态进入系统。

### 4.5 半月报规则验证

验证结果：通过

说明：

1. 已验证 4 月份半月报录入会被拒绝。
2. 系统能按月份限制 report_type 与 report_period 的合法组合。

### 4.6 企业上报与市级审核

验证结果：通过

说明：

1. 企业草稿记录可以提交进入审核流。
2. 市级审核员可查看待审队列。
3. 市级审核员可执行通过和退回操作。

### 4.7 省级审核与退回重提

验证结果：通过

说明：

1. 市级通过后的记录可进入省级审核队列。
2. 省级审核员可执行终审通过和终审退回。
3. 企业侧可看到退回意见。
4. 被省级退回的记录可重新提交回市级审核链路。

## 5. 已验证的核心业务闭环

当前版本已经验证通过的主链路如下：

1. 企业登录。
2. 企业维护基础信息。
3. 企业录入就业/失业数据。
4. 企业提交记录进入市级审核。
5. 市级审核通过后进入省级审核。
6. 省级审核通过或退回。
7. 企业查看退回意见并重新提交。
