## 2. 项目背景

为满足课程作业中对“项目计划、配置管理、变更控制、Agent 编程软件系统”的综合要求，本项目拟建设一个面向企业就业与失业信息采集的轻量级管理系统。系统用于模拟政府或行业主管部门对企业就业、失业相关数据的采集、录入、查询、统计和导出流程，并通过项目全过程文档沉淀体现项目管理与配置管理能力。

本项目既是一个可运行的软件系统，也是一个完整的软件工程过程演练案例。项目不仅要求交付系统本身，还要求提交项目计划、计划迭代历史、变更单等过程性材料，因此项目实施必须采用“文档与系统同步推进”的方式开展。

## 3. 建设目标

### 3.1 总体目标

在 2 周内完成一个可演示、可运行、可提交的软件系统课程作业版本，形成完整的软件交付件与项目管理过程文档。

### 3.2 具体目标

1. 实现企业就业失业数据采集业务的核心闭环。
2. 完成从项目启动、计划制定、任务分工、迭代开发到变更控制的全过程材料沉淀。
3. 通过 Git/GitHub 记录项目计划迭代、代码演进和变更处理历史。
4. 构建适合课堂展示的系统原型与演示路径。

## 4. 用户对象

本项目面向以下三类用户对象进行设计：

1. 系统管理员：负责系统用户、角色权限和基础配置维护。
2. 企业填报人员：负责录入企业基础信息、就业数据和失业数据。
3. 主管部门查看人员：负责查看汇总数据、筛选查询和统计导出。

## 5. 项目范围

### 5.1 本期范围（课程作业版 MVP）

本期仅交付课堂作业版 MVP，范围锁定如下：

1. 用户登录与基础角色权限。
2. 企业基础信息管理。
3. 就业数据录入、编辑、查询。
4. 失业数据录入、编辑、查询。
5. 基础条件筛选与列表展示。
6. 基础统计展示。
7. 数据导出功能。
8. README、运行说明、项目计划、CM 历史、变更单等配套文档。

### 5.2 本期不做内容（明确排除，防止范围膨胀）

以下功能不纳入本期 2 周交付范围：

1. 复杂审批流。
2. 短信、邮件、第三方接口集成。
3. 多租户与多地区分级管理。
4. 高复杂度可视化大屏。
5. 移动端 App。
6. 生产级部署与高并发优化。

### 5.3 范围冻结原则

1. 第 1 天冻结系统核心范围，后续仅允许通过变更单增加有限功能。
2. 新增需求必须说明业务价值、影响范围、工期影响和验证方式。
3. 若新增功能影响主线交付，则优先保证 MVP 核心闭环，不追求大而全。

## 12. 甘特图任务说明

甘特图将以“任务编号 + 起止天数 + 持续时间 + 前置任务 + 责任角色”的形式编制。为便于后续导入 Excel 或项目管理工具，本次单独提供一份任务表文件 [01_Project_Plan/Gantt_Task_Table_v0.2.csv](01_Project_Plan/Gantt_Task_Table_v0.2.csv)。

甘特图编制规则如下：

1. 以天为最小计划单位，覆盖第 1 天至第 14 天。
2. 核心开发任务优先串联，非关键文档类任务可并行执行。
3. 变更任务不打断主线交付，在核心闭环完成后再集中纳入。
4. 文档与配置管理任务贯穿全程，确保每轮迭代均可留痕。
