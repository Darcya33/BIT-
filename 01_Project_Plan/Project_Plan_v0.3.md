# 云南省企业就业失业数据采集系统项目计划书 v0.3

## 1. 文档信息

- 项目名称：云南省企业就业失业数据采集系统
- 文档版本：v0.3
- 文档状态：变更后修订版
- 编制日期：2026-04-14
- 项目经理：你本人（PM）
- 执行方式：PM 统筹 + Agent 协同开发与文档产出
- 修订依据：CR-001、CR-002

## 2. 变更背景

在项目基础计划和系统骨架完成后，项目收到两次正式需求变更：

1. 一季度由月报调整为半月报。
2. 新增企业上报、市级审核、省级审核手机端能力，并将总周期延长两周。

因此，项目计划需要从“短周期演示版排期”调整为“阶段化交付排期”。

## 3. 调整后的项目目标

在保留原有企业就业失业数据采集能力的基础上，项目目标调整为：

1. 支持 1 至 3 月半月报、其余月份整月报。
2. 支持企业上报、市级审核、省级审核完整业务链路。
3. 支持手机端完成填报与审核核心操作。
4. 在项目交付过程中形成完整计划、变更单、CM 历史和源码成果。

## 4. 调整后的范围

### 4.1 本期范围

1. 登录认证与多角色权限。
2. 企业信息管理。
3. 就业和失业数据录入。
4. 半月报与整月报双报表模式。
5. 企业上报流程。
6. 市级审核流程。
7. 省级审核流程。
8. 查询、统计、导出。
9. 手机端适配。
10. 项目文档、变更单、CM 历史、README、演示材料。

### 4.2 仍不纳入本期范围

1. 外部系统集成。
2. 复杂消息通知。
3. 生产级部署与性能优化。
4. 原生移动 App。

## 5. 调整后的角色范围

系统角色调整为：

1. 企业填报人员。
2. 市级审核人员。
3. 省级审核人员。
4. 系统管理员。

## 8. 对现有系统开发的要求

基于当前骨架，后续开发必须遵循以下原则：

1. 数据模型优先支持报表周期和审核状态。
2. 权限模型一次性设计为多角色。
3. 页面结构按桌面端与手机端兼容方式实现。
4. 每轮开发结束后更新文档、清理辅助文件、提交并推送远端仓库。

## 9. 结论

项目需要调整迭代内容，其中：

1. CR-001 属于核心业务规则变更，应立即前置到填报功能开发中。
2. CR-002 属于重大范围和周期变更，应作为总体计划修订依据。
3. 当前系统骨架可继续使用，无需推倒重来。
4. 后续迭代不应再按原 14 天版本推进，而应按 v0.3 的阶段化计划执行。
