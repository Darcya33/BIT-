# 云南省企业就业失业数据采集系统 演示脚本

## 1. 演示目标

本脚本用于课程展示或答辩时快速说明系统背景、角色分工、核心业务流程和当前实现边界。

## 2. 演示前准备

### 后端启动

在仓库根目录执行：

```powershell
Set-Location 04_Source_Code/backend
e:/桌面/云南省企业就业失业数据采集系统/.venv/Scripts/python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

如果 8000 端口被占用，可以改用 8001 或 8004。

### 前端启动

直接打开 frontend/index.html，或使用 Live Server 打开 frontend 目录。

## 4. 推荐演示顺序

### 第一部分：项目背景与系统结构

讲解要点：

1. 本系统用于企业就业失业数据采集。
2. 项目经过需求变更后，支持半月报与多级审核流程。
3. 前端采用原生 HTML/CSS/JS，后端采用 FastAPI + SQLite。

### 第二部分：管理员视角

操作步骤：

1. 使用 admin 账号登录。
2. 展示概览卡片和企业管理页面。
3. 说明管理员可查看系统总体数据并维护企业信息。

讲解要点：

1. 管理员负责配置企业基础信息。
2. 企业信息是后续填报和审核流程的基础。

### 第三部分：企业用户视角

操作步骤：

1. 使用 enterprise_demo 登录。
2. 展示企业信息页。
3. 进入填报页，新增一条就业记录和一条失业记录。
4. 展示 1 至 3 月必须半月报、其他月份按月报的规则。
5. 提交草稿进入审核流程。

讲解要点：

1. 企业先保存草稿，再执行上报。
2. 系统会根据月份自动校验填报周期是否合法。

### 第四部分：市级审核员视角

操作步骤：

1. 使用 city_reviewer 登录。
2. 打开审核队列。
3. 对待审记录执行通过或退回。

讲解要点：

1. 市级审核员负责初审。
2. 通过后记录进入省级审核队列。
3. 退回后企业可根据意见修改再提。

### 第五部分：省级审核员视角

操作步骤：

1. 使用 province_reviewer 登录。
2. 展示省级待审队列。
3. 对一条记录执行终审通过。
4. 对另一条记录执行退回。

讲解要点：

1. 省级审核构成最终审核环节。
2. 终审退回意见会展示给企业，并支持重新提交。

### 第六部分：企业查看退回意见

操作步骤：

1. 切回 enterprise_demo 账号。
2. 打开记录列表。
3. 展示退回意见列。
4. 点击重新提交，将记录重新送入市级审核队列。

讲解要点：

1. 系统已经形成从录入到审核再到退回重提的闭环。
2. 这是当前版本最重要的可演示业务成果。

## 5. 建议答辩话术

1. 项目不是一次性静态开发，而是在计划、变更、实现、验证四条线上同步推进。
2. 两次正式需求变更都形成了文档、计划调整和代码落地记录。
3. 当前版本重点展示的是核心业务闭环与配置管理过程，而非最终生产级完整系统。

## 6. 当前版本边界说明

1. 当前数据库采用 SQLite，主要用于课程演示。
2. 登录鉴权采用演示化简方案，重点突出角色与流程。
3. 查询统计、审核时间线和更多移动端细节仍可继续迭代。
